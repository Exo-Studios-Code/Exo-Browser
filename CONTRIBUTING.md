# Contributing to Ex0-Browser

First off, thank you for considering contributing to Ex0-Browser! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

To maintain a high standard of code quality and ensure efficient development, please review the following guidelines before participating.

---

## 🗺️ Where to Start?

If you have questions, want to discuss architectural changes, or need help with development, join our official community hub:
👉 **Join our Discord:** [https://discord.gg/b5B9tzHNEv]

---

## 🐛 Reporting Bugs

Before opening a new Issue on GitHub, please check the existing Issues to see if the bug has already been reported. If it hasn't, feel free to open a new one using the following structure:

1. **Clear Summary:** Use a descriptive title (e.g., `[Bug] Vertical monitor window scaling issue`).
2. **Steps to Reproduce:** Provide a step-by-step guide on how to trigger the bug.
3. **Expected vs. Actual Behavior:** What should have happened versus what actually happened.
4. **Environment Details:** Include your Operating System, hardware, and runtime environment if applicable.
5. **Logs/Screenshots:** Attach any relevant error logs or visual proof.

---

## 💡 Suggesting Features

We highly value innovative ideas! If you want to propose a new feature or improvement:
* Clearly explain the problem this feature solves and the value it adds to power users.
* Feel free to submit feature ideas in our Discord channel `<#feature-requests>` for immediate community feedback before drafting a full specification.

---

## 💻 Pull Requests (PR) Workflow

If you want to contribute code directly, please follow this process to get your PR merged:

1. **Fork the Repository:** Create your own fork and clone it locally.
2. **Create a Feature Branch:** Use descriptive names (e.g., `feature/custom-widgets` or `fix/cors-error`).
3. **Keep Code Clean:** Follow standard coding practices for JavaScript/Node.js, write clear commit messages, and ensure no breaking changes are introduced.
4. **Submit the PR:** Fill out the PR template clearly describing what changes were made and why.

---

## 📜 Code of Conduct

By participating in this project, you agree to abide by our server rules and maintain a professional, respectful, and constructive environment for all maintainers and contributors. 

*Thank you for helping us build the future of Ex0 Studios!*
