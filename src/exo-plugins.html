Developer Guide — How to write a plugin ▾

A plugin is a folder with the following files, packed into a .zip:

manifest.json

{
  "id":          "my-plugin",
  "name":        "My Plugin",
  "version":     "1.0.0",
  "description": "What this plugin does",
  "author":      "Your Name",
  "permissions": ["storage", "notifications"],
  "content_scripts": {
    "matches": ["*://youtube.com/*", "*://*.github.com/*"],
    "js":      "content.js",
    "css":     "styles.css"
  },
  "toolbar_action": {
    "icon":    "icon.svg",
    "tooltip": "Run plugin"
  }
}

content.js — available API

// window.exo is available in the content script

// Logging
exo.log('Hello from page', location.href);

// Storage (localStorage scoped to the plugin namespace)
exo.storage.set('counter', 42);
const c = exo.storage.get('counter');  // → 42

// Notifications
exo.notify('Plugin active', 'Page has been loaded.');

// Clipboard
exo.clipboard.write('copied text');

// Plugin info
console.log(exo.plugin.name, exo.plugin.version);

// Regular DOM manipulation
document.querySelectorAll('.ad').forEach(el => el.remove());

URL Patterns (matches)

"*://youtube.com/*"        — YouTube only
"*://*.github.com/*"       — GitHub and subdomains
"https://*/path/*"         — HTTPS with /path/ only
"<all_urls>"               — all pages
